**Claude Code核心运行模型AGENTIC LOOP**
* **运行逻辑**：核心是三步循环的运行模式，类似侦探破案的流程。

* **第三步**：验证结果，包括运行测试、检查输出、审查代码变更。

* **context surface**：负责决定哪些信息常驻上下文，哪些信息需要按需加载。
* **action surface**：负责决定Claude Code可以执行哪些操作。
* **control surface**：负责约束必须被管控的操作。
* **isolation surface**：负责决定哪些任务需要隔离上下文。
* **verification surface**：负责判断任务是否真正完成。

**hooks层核心特性与优势**
* **规则执行特性**：普通配置规则只是给Claude的建议，Claude不一定遵守，而hooks是硬编码的拦截器。
